Ansible and AWS

Gaurav Bhatt

Configuration is a powerful tool used by automation frameworks to hide code complexity and thus enhance productivity. In the world of cloud computing and IaC (Infrastructure as Code) tools like Ansible help a cloud engineer to perform complex deployments by wrapping multiple complex low level API calls under a simple yaml configuration. In this article, I am going to show how we can use Ansible to create resources in AWS and manage them.

Lets understand the folder structure:

  • ansible: This is the root folder of the project.
  • inventory: Keep config files containing information about the machines where deployments will take place.
  • roles: A way to group tasks together.
  • deploy: Logical action to group tasks which are part of an important step in Application's continuous delivery. e.g. The deployment of a web application which contains tasks such as creating database stack and web stack.
  • files: Folder to keep cloud formation templates for different application stacks like creation of all IAM policies, EC2 instances etc.
  • tasks: Ansible modules or commands which will perform some action like file copy, AWS stack creation etc.
  1. Create 'dev' file inside 'ansible/inventory' folder as below:



    [aws]     : Name of the configuration used to refer by ansible configuration which points to ip address / dns name of the hosts machines where the script will be executed.
    ansible_connection: This will let ansible know that connection with the host where the script will execute is 'local'. Other possible values could be 'ansible_connection=ssh'.
    ansible_python_interpreter: To tell the version of python to be used.

  2. Create 'cf-stacks.yaml' under 'ansible' folder with the following code:
    ---
    - hosts: aws
      roles:
        - deploy
    hosts: This points to the machines declared in the inventory which will execute the ansible module/tasks which in this case will be 'localhost'.
    roles: This is used to indicate the folder to look for which would contain tasks directory with main.yaml file.

  3. Create the cloudformation template 's3.yaml' inside 'ansible/roles/deploy/files'.
    AWSTemplateFormatVersion: "2010-09-09"
    Description: 'S3 Stack'
    Resources:
      MainBucket:
        Type: "AWS::S3::Bucket"
        Properties:
          BucketName: 'lh-gb-test'
  4. Create 's3.yaml' under 'ansible/roles/deploy/tasks'.
    ---
    - name: Creating CFN Stack
      cloudformation:
        stack_name: "my-app-stack"
        template: "roles/deploy/files/s3.yaml"
        state: 'absent'
      register: logs
    - debug:
        var: logs
  5. Create 'main.yaml' inside 'ansible/roles/deploy/tasks'.
    ---
    - name: Deploy S3
      include: s3.yaml
      tags:
        s3
    Once above files have been created, run the following command under directory 'ansible'
    ansible-playbook -i inventory/dev cf-stacks.yaml --tags s3
    Once the command completes successfully you can go to S3 web console and should be able to see a new bucket created.
