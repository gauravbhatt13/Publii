Environment variables in Java

Gaurav Bhatt

Environment variables are used to configure/customize operating system user's runtime environment.  They exist outside of any application code and serve as a tool for customization. A very common example is the variable 'PATH' which determines the locations to be used by the system to look for any executables. Following example shows how to retrieve an environment variable using System class.

public class EnvUtil {
    public static void main(String[] args) {
        System.out.println("System.getenv(\"PATH\") = " + System.getenv("PATH"));
    }
}

The above code would produce output similar to:

System.getenv("PATH") = /opt/openjdk-16/bi../Playground/:../Playground/:/usr/sbi../Playground/:/sbi../Playground/

Getting System Properties

Properties are nothing but key value pairs where keys are used to retrieve the values. Java provides java.util.Properties class to manage properties which provides methods to load, retrieve, list, enumerate and save properties. Java Platform System class uses Properties class to store its own configuration of the current working environment.

The System class has two methods used to read system properties: getProperty and getProperties.

There are two versions of getProperty , one which accepts only the key and the other which accepts a key and a default value to be returned when key is not present.

System.getProperty("path.separator");
System.getProperty("path.separator", ":");

The System class uses a static Properties class instance which keeps following properties:

  1. Operating system properties like 'os.name'
  2. System and user properties like 'java.class.path'
  3. Properties defined on the command line using -D

Pass Environment Variable config using command line

The -D option can be used to pass a key value pair to an application program in Java.

java -D"version=1.2" MyApplication
