Difference between Root user and IAM user

Amazon Web Services require a central management system to manage all the users and the access rules. It would have been a nightmare for Amazon as well its customers if every service would provide its own access and management solution. IAM thus becomes a very crucial service and it becomes essential for the customers of AWS to understand how this whole setup works.

Types of Users

There are two distinct users on a very high level:

  1. Root user
  2. IAM user

Root user

To create an account with AWS, we need to provide an email address which is used to create a sign-in identity which is called as Root user. To sign in as a root user, you need to use the email address and password used while creating the AWS account. 

 

IAM user

Any user that is created using IAM console will be called as IAM User. Note the difference that the root user was never created using IAM
