Java and Docker development environment

Gaurav Bhatt

Running docker image as a container isolates the application within the image from the host environment where the docker daemon is running, which fastens the processing and enhances productivity of the developer. A completely isolated environment can be used to quickly spin up heavy softwares like databases and middle wares for integration testing and proof of concepts.

Demo

Steps

  • Create a Gradle project with following configuration:
    plugins {
        id 'java'
    }

    ext {
        javaMainClass = 'tutorial.App'
    }

    group 'org.example'
    version '1.0-SNAPSHOT'

    repositories {
        mavenCentral()
    }

    dependencies {
        testCompile group: 'junit', name: 'junit', version: '4.12'
    }

    jar {
        manifest {
            attributes ("Main-Class": javaMainClass)
        }
    }
  • Create a package called tutorial and add new java file called App.java with following code:
    package tutorial;

    import java.io.IOException;
    import java.net.ServerSocket;
    import java.net.Socket;

    public class App {
        public static void main(String[] args) throws IOException {
            ServerSocket serverSocket = new ServerSocket(8081);
            Socket acceptedSocket = serverSocket.accept();
            int data;
            while((data = acceptedSocket.getInputStream().read()) != -1) {
                System.out.println((char)data);
            }
        }
    }
  • Create another java file called Client.java with the following code:
    package tutorial;

    import java.io.IOException;
    import java.net.Socket;

    public class Client {
        public static void main(String[] args) throws IOException {
            Socket s = new Socket("127.0.0.1", 8081);
            s.getOutputStream().write(65);
            s.close();
        }
    }
  • Run App.java which will start a TCP server and then run Client.java which will connect to that server and send character 'A' which will then be received by the server and printed on the screen. This will ensure that the code we ran runs without docker.

  • Now we will create a folder called docker in the project and create a new file called 'Dockerfile' with the following content:
    FROM openjdk
    COPY libs/docker-test-1.0-SNAPSHOT.jar /
  • Run ./gradlew jar task which will generate 'docker-test-1.0-SNAPSHOT.jar' file inside build/libs folder.
  • Navigate to build folder in the project and run the following command:
    docker build -f ../docker/DockerFile -t test-docker:1.0 .
  • This will build the docker image with our application inside. Now run the container for the image by the following command:
    docker run -p 8081:8081 test-docker:1.0 java -jar docker-test-1.0-SNAPSHOT.jar
  • This will start our container with the server. Now we can run Client.java again to test and see the output of the container.
