GIT Error: src refspec main does not match any: Solved

GIT default branch name earlier was 'master' and in order to move the tech industry to a more inclusive, open culture - and removing language like "master/slave", alternate options popped up suggesting default/primary as on option which eventually settled on naming default directories as 'main'. This caused a very common issue 'src refspec main does not match' due to the fact that when we go to Github and create a new repository, it now names it as 'main' instead of 'master' as shown in below images,  while when we do 'git init' in our local machine and commit, it will create a branch named 'master'. So when we try to push that branch to Github it complains with the error 'src refspec main does not match'. Following image show step by step order to recreate the error.

SOLUTION

Following steps would help in resolving the issue:

Rename the current branch to main

git branch -m master main

 

Once the local branch is renamed try the git push -u origin main again and you should see the upload.



Configure GIT default directory

Also make sure that we change the git configuration to configure default directory as follows:

git config --global init.defaultBranch main

The above command will make sure that every next time you create a new git repository locally, it will create default branch with 'main' as the branch name.
